# قرارات التنفيذ — أثر

> ملخّص قرارات مثبّتة بعد جلسة غرّيل (٢٠٢٦/٠٦/٢٨). يكمّل وثيقة البنية التقنية (`12`) ويحسم قراراتها المفتوحة.

## الرهان الأساسي
بناء كل الـPRD مباشرة — **بدون مرحلة تحقّق مسبقة**، بالتوازي، بفريق من اثنين، في ~٣ أشهر.

- **الخطر المقبول:** اكتشاف مشاكل الجودة/الدفع متأخّراً (بعد البناء) بدل كشفها مبكّراً عبر concierge بأسبوعين.
- **القرار اتُّخذ بوعي** رغم تحذير الدراسة المتكرّر من البناء قبل التحقّق.

## الشريحة ومعيار النجاح
- **النيّش:** خدمات مهنية/B2B سعودية نشطة فعلاً على لينكدإن + X (حسابات نشطة فقط).
- **معيار الجودة:** عيّنة أثر تُقارَن بـ(أ) محتوى العميل الحالي + (ب) مخرجات ChatGPT/Claude خام لنفس الشركة.
- **اختبار الدفع:** تجربة مدفوعة حقيقية (شهر مخفّض/دفعة)، لا سؤال مجاني.
- **عتبة التوسّع:** ≥٣ يدفعون من الدفعة الأولى + **تجديد الشهر الثاني** كبوابة ثانية قبل ضخّ التسويق.
- **مصدر العملاء الأوائل:** الشبكة الشخصية.

## قرارات البناء

### ١. الاستضافة
multi-tenant منطقي (tenantId + تشفير + عزل صلاحيات). يكفي PDPL. تأجيل per-customer حتى ٣+ عملاء (نهج CareKit).

### ٢. الصور
- هوية الشركة (ألوان، شعار، ستايل) تُهيّأ وتُخزّن في الـsetup مرة واحدة.
- التوليد عبر **gpt-image** بالنص، بهوية الشركة محقونة في البرومبت.
- **حلقة تحقّق بصري:** نموذج رؤية يقرأ الصورة، يستخرج النص العربي، ويقارنه بالمقصود → تطابق = اعتماد؛ خطأ/تكسّر = إعادة توليد (سقف ٢–٣)؛ فشل = إسقاط على overlay برمجي للنص.
- يضيف OpenAI كمزوّد ثانٍ جنب Claude (نص=Claude، صورة=GPT)؛ ترصد تكلفة الصورة في اقتصاد الوحدة.
- **مهمة بناء #١:** اختبار ٢٠ صورة بنصوص عربية واقعية ببرومبت معزّز لقياس نسبة التكسّر قبل اعتماد المعمارية نهائياً. تكسّر <١٠٪ = gpt-image للنص + تحقّق؛ مرتفع = overlay أساساً.

### ٣. البحث والمصادر
- العميل يحدّد **محاوره ومواضيعه** في الـsetup، والنظام يقترح محاور مناسبة لمجاله؛ تُخزّن في دماغ الشركة. العميل يقرّر، الذكاء ينفّذ.
- آلية الجلب: **بحث حيّ مقيّد بمصادر موثوقة، موجّه بمحاور العميل** → بوست بمصدر (URL) لكل معلومة فعلية.
- **لا RAG كامل** — المحاور مفتوحة وغير متوقّعة لكل عميل، فمخزن ثابت لا يغطّيها ويتقادم.

### ٤. التذكيرات
داخل التطبيق + بريد للإطلاق. واتساب fast-follow بعد استقرار مزوّد BSP (تجنّب تعليق الإطلاق على اعتماد طرف ثالث).

### ٥. الفريق والجدول
- **علي:** frontend — Next.js RTL، onboarding، محرّر البوست، التقويم، النشر اليدوي.
- **أنت:** backend/المحرّك — Claude، البحث، خط أنابيب الصور + التحقّق، jobs/queue، Moyasar، multi-tenant.
- فصل بالطبقة بعقد API واضح بينكم (يتجنّب التصادم على الطبقات المشتركة).
- **الجدول الواقعي:** ~٣ أشهر لإطلاق صلب (يطابق خارطة الطريق: شهر ٣ = إطلاق).
