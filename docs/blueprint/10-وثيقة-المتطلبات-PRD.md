# وثيقة المتطلبات (PRD) — أثر

## نظرة عامة
"أثر" منصّة SaaS عربية تنتج محتوى سوشيال ميديا للشركات الصغيرة والمتوسطة السعودية: تهيئة عميقة لكل شركة (company brain)، محرّك بحث→بوست بمصادر، كتابة عربية بنبرة العلامة، توليد صور بنص عربي، وتقويم/تخطيط نشر. **النشر في النسخة الأولى يدوي مُيسّر** (لا API نشر).

## المشكلة والأهداف
الـSMB يحتاج محتوى منتظم وجيد بلا فريق أو وكالة غالية. أهداف المنتج (نتائج قابلة للقياس):
- إنتاج العميل لـ"شهر محتوى" جاهز في **<٣٠ دقيقة** من بدء التهيئة.
- نسبة بوستات تُعتمد بلا تعديل جوهري **≥٦٠٪** (مؤشر جودة).
- احتفاظ شهر ٣ **≥٧٠٪** (مؤشر قيمة).

## النطاق

**داخل النطاق (هذه النسخة):**
- تسجيل/دخول وإدارة حساب واشتراك.
- تهيئة ذكية: إدخال رابط الموقع/الحسابات، تحليل آلي للمحتوى السابق، أسئلة تأكيد قصيرة → بناء `BrandProfile`.
- محرّك بحث→بوست: جمع معلومة من الويب + توليد بوست عربي + **اقتران كل معلومة بمصدر**.
- كتابة بنبرة العلامة، وصياغة مختلفة لـLinkedIn مقابل X.
- توليد صور بنص عربي مدموج عبر gpt-image + تحقّق بصري (overlay احتياطي عند الفشل).
- تقويم محتوى سعودي (مناسبات/مواسم) + تذكيرات.
- سير اعتماد (draft → review → approved).
- **نشر يدوي مُيسّر:** نسخ منسّق بضغطة، تنزيل الصورة، فتح المنصة (deep link حيث يتاح)، تذكير مجدول.
- دفع Moyasar + إدارة الباقة.
- دعم RTL/عربي كامل + موبايل-أول.

**خارج النطاق (مؤجل):**
- النشر التلقائي عبر LinkedIn/X APIs.
- التحليلات وقياس الأداء (تحتاج وصول API).
- ردود تلقائية على التعليقات/الرسائل.
- تعدّد العملاء/العلامات، White-label، مقاعد فريق متقدمة.
- منصات غير LinkedIn/X، الفيديو.
- حلقة التعلّم الذاتي من الأداء.

## المتطلبات الوظيفية (Functional Requirements)

**التهيئة (Onboarding / Brand Brain)**
- **FR-1:** يقبل النظام رابط موقع العميل وروابط/معرّفات حساباته، ويحلّلها لاستخلاص نبرة، منتجات/خدمات، جمهور، كلمات مفتاحية.
- **FR-2:** يطرح النظام أسئلة تأكيد قصيرة (نبرة، ممنوعات/محظورات، منافسون، أهداف)، ويتيح للعميل تحديد **محاوره/مواضيعه** للبحث مع **اقتراحات محاور مناسبة من النظام**، ويبني `BrandProfile` قابل للتعديل (شاملاً المحاور المعتمدة).
- **FR-3:** يخزّن النظام `BrandProfile` (بما فيه المحاور المعتمدة) كمرجع/سياق لكل توليد لاحق.

**محرّك المحتوى**
- **FR-4:** يولّد النظام بوستات عربية بناءً على `BrandProfile` + موضوع/توجيه، بصياغة مخصّصة لكل منصة (LinkedIn/X).
- **FR-5:** عند توليد محتوى يتضمّن معلومة فعلية (إحصائية/خبر/سياق)، يجلب النظام المعلومة عبر **بحث حيّ مقيّد بمصادر موثوقة داخل محاور العميل** (المحدّدة في التهيئة بـ`BrandProfile`)، **ويرفق المصدر (URL) لكل معلومة فعلية مع البوست**. (لا RAG — المحاور مفتوحة لكل عميل.)
- **FR-6:** يولّد النظام صورة مرافقة عبر **gpt-image (OpenAI) مع دمج النص العربي داخل الصورة**، مستنداً إلى هوية الشركة المخزّنة في التهيئة، ثم تمرّ بـ**حلقة تحقّق بصري** (نموذج رؤية يقرأ النص من الصورة ويقارنه بالمقصود؛ تطابق = اعتماد، خطأ = إعادة توليد ٢–٣ مرات، فشل = إسقاط على **overlay برمجي احتياطي** فوق القالب).
- **FR-7:** يولّد النظام "خطة شهر" (مجموعة مسودات موزّعة على التقويم) دفعة واحدة.

**التقويم والاعتماد**
- **FR-8:** يعرض النظام تقويم محتوى يقترح مواعيد ومناسبات سعودية (وطنية/مواسم/رمضان).
- **FR-9:** يمرّ كل بوست بحالات: `draft → pending_review → approved`، ويسمح بالتعديل قبل الاعتماد.
- **FR-10:** يتيح النظام جدولة "تذكير نشر" لكل بوست في الوقت المقترح.

**النشر اليدوي المُيسّر**
- **FR-11:** لكل بوست معتمد، يوفّر النظام: نسخ النص المنسّق بضغطة، تنزيل الصورة، وزر "افتح المنصة" (deep link / web intent حيث تسمح المنصة).
- **FR-12:** يرسل النظام تذكيراً عند موعد النشر مع البوست جاهزاً عبر **قناتي الإطلاق الأساسيتين: داخل التطبيق + بريد إلكتروني**؛ واتساب لاحقاً (fast-follow).
- **FR-13:** يتيح النظام تعليم البوست "تم نشره" يدوياً لتتبّع الحالة.

**الحساب والاشتراك**
- **FR-14:** تسجيل/دخول، إدارة ملف الشركة، إدارة عدة "ملفات حسابات" (للتخصيص، بلا ربط API) ضمن العميل الواحد.
- **FR-15:** اشتراك ودفع عبر Moyasar، تطبيق سقوف الباقة (مسودات/صور شهرياً)، وتجربة مجانية ٧ أيام.

## المتطلبات غير الوظيفية (Non-Functional Requirements)
- **NFR-1 (RTL/عربي):** دعم عربي كامل RTL في كل الواجهات والمخرجات؛ خط واضح (مثل IBM Plex Sans Arabic).
- **NFR-2 (الأداء):** توليد بوست مفرد خلال ثوانٍ معقولة؛ توليد "خطة شهر" غير متزامن (async job) مع مؤشّر تقدّم.
- **NFR-3 (الموبايل):** تجربة موبايل-أول (الـSMB يدير من جواله).
- **NFR-4 (الأمان):** تشفير البيانات، إدارة أسرار آمنة، فصل بيانات كل عميل (tenant isolation منطقي).
- **NFR-5 (الامتثال):** توافق PDPL (موافقة، تقليل بيانات، حق الحذف)، وفاتورة اشتراك بسيطة، وأساس بيانات داخل نطاق مناسب.
- **NFR-6 (الجودة/الأمان التحريري):** كل معلومة فعلية مقرونة بمصدر؛ لا نشر بلا اعتماد بشري في هذه النسخة؛ وتمرّ الصور بـ**تحقّق بصري** (قراءة النص ومطابقته بالمقصود) قبل العرض على العميل.
- **NFR-7 (التكلفة):** تتبّع استهلاك الـAI لكل عميل لمراقبة الهامش؛ سقوف قابلة للضبط.
- **NFR-8 (التوسّع):** فصل المحرّك خلف `ContentProvider` interface لتبديل/اختبار نماذج لاحقاً.

## القيود والاعتمادات
- Claude API (محرّك التوليد) — مع abstraction للتبديل.
- OpenAI / gpt-image (مزوّد ثانٍ جنب Claude) — توليد الصور بالنص العربي المدموج.
- نموذج رؤية للتحقّق البصري + مكتبة طباعة نص عربي (overlay) كـ**احتياط** عند فشل التحقّق (لا كأساس).
- Moyasar (مدفوعات).
- مصدر بحث ويب (web search/fetch) — **بحث حيّ مقيّد بمصادر موثوقة داخل محاور العميل** للمعلومة والمصادر.
- **اعتماد مؤجّل:** LinkedIn Marketing Developer Platform و X API (للنشر التلقائي في V2) — موافقة الشريك وتكاليف pay-per-use.

## معايير القبول العامة للنسخة
- عميل جديد يكمل التهيئة ويحصل على "خطة شهر" مسودات جاهزة.
- كل بوست يحوي معلومة فعلية يعرض مصدرها.
- الصور تظهر النص العربي سليماً (بلا تكسّر حروف)، بعد اجتياز التحقّق البصري، مع إسقاط على overlay احتياطي عند الفشل.
- مسار النشر اليدوي يعمل (نسخ/تنزيل/فتح/تذكير) على الموبايل.
- الاشتراك والدفع عبر Moyasar يعملان، والسقوف مطبّقة.
