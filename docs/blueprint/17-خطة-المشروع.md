# خطة المشروع على مراحل — أثر

> خريطة المراحل (plan-of-plans). الطريقة: كل مرحلة لها دورتها الكاملة **spec → plan → build** بمنهجية superpowers، ننفّذها مرحلة-مرحلة ونحن في الحلقة — لا خطة عملاقة واحدة تتقادم.
>
> backend = طارق · frontend = علي · multi-tenant منطقي · النشر يدوي · لا أتمتة/تحليلات (V2).

## المبدأ
- كل مرحلة تُسلّم **برمجية تشتغل وتُختبر وحدها**.
- لكل مرحلة: نكتب الـspec (إن لم يوجد) → نحوّله plan تفصيلي (writing-plans) → ننفّذ (subagent-driven-development) بمراجعة بين المهام.
- لا نكتب خطط المراحل القادمة مسبقاً — نكتب خطة المرحلة عند بدئها لتعكس ما تعلّمناه.
