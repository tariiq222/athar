# المرحلة ٧ — الواجهة الأمامية (Frontend Spec) — أثر

> واجهة العميل الكاملة لمنصّة "أثر" (محتوى عربي). تستهلك خدمات الـbackend عبر عقود REST (NestJS) وتعرضها بتجربة **عربية-أوّل، RTL كامل، موبايل-أوّل**. لا منطق أعمال في الواجهة — التوليد والبحث والفوترة والاستشهاد كلّها خلف الـAPI. تغطّي هذه المرحلة شاشات المراحل ١–٦ بصرياً وتربطها بعقودها. تطبّق حدود المنصات من [15-مواصفات-المنصات.md](../../blueprint/15-مواصفات-المنصات.md) مرئياً عبر `CharCounter`، وتعرض مخرجات محرّك [16-معمارية-المحرّك.md](../../blueprint/16-معمارية-المحرّك.md).
>
> **الإصدار:** V1 (منقّح — نطاق مخفَّض، M2 هو المنتج الحقيقي القابل للتجربة). المنفّذ: Ali. التسليمات الكاملة في القسم ١١.
>
> **Design Context:** [.impeccable.md](../../.impeccable.md) هو الـsource of truth لكل القرارات البصرية. هذا الـspec يستهلكه ولا يعيد تعريفه.

## ١. المبادئ

- **عربي-أوّل، RTL كامل:** `dir="rtl"` و`lang="ar"` على الجذر؛ كل تخطيط يُصمّم RTL أصالةً لا كقلب لاحق. خط `IBM Plex Sans Arabic` للجسم، `TheYearofHandicrafts` للعناوين display، `Effra` للـLatin — كلّها في `app/fonts/` داخل المشروع (لا مسارات من جهاز المطوّر).
- **موبايل-أوّل:** التصميم يبدأ من عرض الموبايل ثم يتوسّع؛ أهداف اللمس ≥٤٤px، تنقّل سفلي على الموبايل.
- **لا منطق أعمال في الواجهة:** الواجهة تعرض وتجمع المدخلات وتستدعي العقود فقط. كلّ الحدود والقيود خلف الـbackend.
- **TanStack Query لطبقة البيانات:** استعلام + mutation + caching + invalidation بدون طبقات `services/` و`hooks/` إجبارية. Service wrapper فقط عند الحاجة (انظر القسم ٥).
- **حالات صريحة دائماً:** كل شاشة تعرّف loading/empty/error/success.
- **StatusBadge = المسار الوحيد لعرض الحالة.**
- **Premium without ornament:** نعكس Linear/Arc/Vercel. لا gradient text، لا side-stripe borders، لا glassmorphism، لا hero-metric template.

## ٢. التقنية والبنية

- **Next.js 15 (App Router) + React 19 + TypeScript.**
- **التنسيق:** CSS Modules + global tokens في `styles/tokens.css` (OKLCH variables). لا utility-class framework — لا Tailwind. (أزلنا Tailwind بالكامل لتجنّب التشتيت.)
- **الخطوط:** ملفّات في `app/fonts/`:
  - `app/fonts/handicrafts/` (TheYearofHandicrafts TTF/OTF — Reg, Med, SemiBold, Bold, Black)
  - `app/fonts/plex-arabic/` (IBM Plex Sans Arabic — عبر next/font/local)
  - `app/fonts/effra/` (Effra — Reg, Med, SemiBold, Bold + italics + Std)
  - **الترخيص:** TheYearofHandicrafts و Effra يجب التحقق من ترخيصهما قبل التضمين في production. في الـM1 يُضافان كـ assets مع TODO لتعليق الترخيص.
- **الحالة:** TanStack Query لـserver state. Contexts خفيفة: `SessionContext`, `LocaleContext`, `ToastContext`.
- **Auth:** Cookie httpOnly فقط (`SameSite=Lax`, `Secure` في production). الـfrontend يستخدم `credentials: 'include'` ولا يحقن Bearer headers.

### ٢.٢ تدفّق البيانات (TanStack Query فقط)

```
Component  →  useQuery / useMutation (queries/usePosts.ts, ...)
   ↓
Hook       →  fetch('/api/v1/...', { credentials: 'include' })
   ↓
fetch      →  /api/v1/*  (NestJS)
```

- **`apiClient` هو fetch wrapper بسيط** يحقن `credentials: 'include'` ويعالج 401 (logout + redirect `/login` مع حفظ المسار).
- **لا طبقات services/hooks إلزامية.** إذا احتاج endpoint منطقاً مركّباً (مثل `startMonthPlan`)، يُغلَّف في hook مخصّص داخل `queries/`. خلاف ذلك، `useQuery` يكفي.
- **الاستثناء الوحيد:** `apiClient` نفسه لأن 401 handling و base URL و envelope normalization مركزية.

## ٣. Design System (المرجع: [.impeccable.md](../../.impeccable.md))

### ٣.١ الألوان (OKLCH tokens في `styles/tokens.css`)

نفس الجدول من النسخة السابقة — primary-deep، primary، surface (cream)، accent (terracotta)، border، heading، mint، sage، muted، elevated. **القاعدة 60-30-10 + ممنوعات ملزمة** سارية.

### ٣.٢ Typography

TheYearofHandicrafts (display) + IBM Plex Sans Arabic (body) + Effra (Latin). 7-step scale، `tnum` للأرقام.

### ٣.٣ Spacing

4pt scale (4/8/12/16/24/32/48/64/96) semantic tokens.

### ٣.٤ Motion

5 timing tokens + ease-out-expo + reduced-motion.

### ٣.٥ المكوّنات (3 طبقات — راجع [.impeccable.md](../../.impeccable.md) § Component Governance)

**`components/ui/`** — عناصر محايدة المجال (tokens فقط، لا raw colors، API موحَّد):
- `Button` (variants: primary / secondary / tertiary / danger — sizes: sm / md / lg — props: loading, disabled, iconStart, iconEnd)
- `IconButton` (props: label, icon, variant, size, loading, disabled)
- `Input`, `Textarea`, `Select`, `Checkbox`, `RadioGroup`, `Switch`
- `FormField` (label, hint, error, required, children — يدعم `aria-describedby` على كل input)
- `Modal`, `Dialog`, `DropdownMenu`, `Popover`, `Tooltip`, `Tabs`, `Pagination`, `Spinner`, `Divider`

**`components/shared/`** — مرتبطة بأثر، تُستخدم في أكثر من شاشة:
- `StatusBadge` (المسار الوحيد لعرض حالة البوست، 4 حالات + `scheduled`/`failed` مشتقّتان)
- `PostCard`, `PlatformBadge`, `PageHeader`, `FilterBar`
- `SourceList` (flat list، روابط `rel=noopener`، فراغ → لا تُعرض)
- `ImagePreview` (overlay fallback label، لا "broken image" placeholder)
- `CharCounter` (LinkedIn ٣٬٠٠٠؛ X بـ twitter-text لا `.length`)
- `EmptyState`, `ErrorState`, `Skeleton`, `Toast`

**`components/layout/`** — هيكل التطبيق:
- `AppShell` (لكل صفحات `(app)`)
- `Header`, `BottomNav` (mobile ≥ 4 وجهات)، `SideNav` (≥md)
- `PageContainer`, `ContentSection`

**قاعدة المكوّنات:**
- ممنوع بناء Button/Input/Modal محلي داخل صفحة.
- `lucide-react` هي مكتبة الأيقونات الوحيدة. strokeWidth = 1.5. أحجام: 16px (نصوص)، 20px (أزرار)، 24px (تنقّل/عناوين).
- أيقونة منفردة بدون `aria-label` ممنوعة.
- الأيقونات الاتجاهية فقط تُعكس في RTL (سهام، إرسال). غير الاتجاهية لا تُعكس (بحث، تقويم، إعدادات، حذف، نسخ).
- أي مكوّن يستخدم في مكانين بنفس السلوك → يُنقل إلى `shared/`.

## ٥. استهلاك الـAPI

```ts
// lib/apiClient.ts — طبقة نقل بيانات فقط، لا redirect
export class UnauthorizedError extends Error { constructor() { super('unauthorized') } }

export const apiClient = {
  async get<T>(path: string): Promise<T> {
    const res = await fetch(`${API_BASE}${path}`, { credentials: 'include' })
    if (res.status === 401) throw new UnauthorizedError()
    if (!res.ok) throw await toApiError(res)
    return res.json() as Promise<T>
  },
  // post/patch/delete: نفس النمط، مع X-CSRF-Token header على mutations
}
```

**`SessionProvider` يلتقط `UnauthorizedError`:** يمسح TanStack cache → redirect إلى `/login?returnTo=<current-path>`. لا يحدث أي redirect داخل `apiClient` نفسه (يبقى SSR-safe وقابل للاختبار).

### استعلامات V1 (TanStack Query)

| Endpoint | Query key | Hook |
|---|---|---|
| `POST /auth/login` | — | `useLogin()` (mutation) |
| `POST /auth/register` | — | `useRegister()` (mutation) |
| `POST /auth/logout` | — | `useLogout()` (mutation) |
| `POST /brand/analyze` | — | `useAnalyzeBrand()` (mutation) |
| `GET /brand/profile` | `['brand', 'profile']` | `useBrandProfile()` |
| `POST /brand/profile` | — | `useCreateProfile()` (mutation) |
| `PATCH /brand/profile/:id` | — | `useUpdateProfile()` (mutation) |
| `GET /posts` | `['posts', filters]` | `usePosts(filters)` |
| `GET /posts/:id` | `['post', id]` | `usePost(id)` |
| `PATCH /posts/:id` | — | `useUpdatePost()` (mutation) |
| `POST /content/generate` (engine) | — | `useGenerate()` (mutation) |
| `POST /content/month-plan` | — | `useStartMonthPlan()` (mutation) |
| `GET /content/month-plan/:jobId` | `['month-plan', jobId]` | `useMonthPlan(jobId)` (polling) |

> كل mutation يبطل الـquery keys المرتبطة (e.g. `useUpdatePost` → `['post', id]` + `['posts']`).

### Auth (Cookie httpOnly + CSRF)

- **لا JWT في localStorage. لا Bearer headers.** Cookie httpOnly فقط.
- `apiClient` يستخدم `credentials: 'include'` على كل طلب.
- `useAuth` يقرأ cookie state عبر endpoint آمن `GET /auth/me` (العقد الكامل في القسم ١١).
- 401 → `apiClient` يرمي `UnauthorizedError` → `SessionProvider` يمسح TanStack cache → redirect إلى `/login?returnTo=<current-path>`.
- آلية الجلسة: cookie session مع access + refresh كزوج cookies، rotation عبر `POST /auth/refresh`.
- **Route UX guard** في Next: middleware يعيد التوجيه إلى `/login?returnTo=...` عند غياب الـcookie. هذه حماية UX فقط، ليست مصدر الحقيقة.
- **API authorization guard** الفعلي في NestJS على كل endpoint tenant-scoped (مصدر الحقيقة).

**حماية CSRF إلزامية (لا استثناءات):**
- **double-submit cookie pattern:** الـbackend يُولّد CSRF token يُخزَّن في cookie غير-`httpOnly` يُقرأه الـfrontend ويضعه في header `X-CSRF-Token` على كل mutation (`POST`/`PATCH`/`DELETE`). الـbackend يتحقّق من تطابق الكوكي والهيدر.
- **Origin/Referer validation:** الـbackend يرفض أي mutation لا يطابق origin قائمة بيضاء (دائماً، بصرف النظر عن deployment topology).
- **CORS صارم:** `Access-Control-Allow-Credentials: true` مع `Access-Control-Allow-Origin` = origin واحد فقط (لا wildcard). قائمة الـorigins من env (`FRONTEND_ORIGINS`).
- **`apiClient` يحقن `X-CSRF-Token` تلقائياً** من cookie `csrf` على كل mutation.

`SameSite=Lax` يقلّل المخاطر لكنه ليس بديلاً عن CSRF token أو Origin validation. هذي متطلبات ثابتة بغض النظر عن deployment topology.

## ٨. إمكانية الوصول (WCAG 2.1 AA + AAA انتقائي)

**V1 = AA إلزامي + AAA للحالات الحرجة:**

| الموضع | المستوى | السبب |
|---|---|---|
| جسم البوست المعتمد | AAA | المحتوى الذي ينشره المستخدم |
| مبالغ وتواريخ الفوترة | AAA | دقة مالية |
| `StatusBadge` labels | AAA | قرار عملي بناءً على الحالة |
| رسائل الخطأ | AAA | المستخدم يتخذ إجراء |
| باقي النصوص | AA | افتراضي V1 |
| `prefers-reduced-motion` | AA | honor دائماً |

**القواعد الملزمة (AA):**
- Contrast 4.5:1 normal text، 3:1 large.
- Color ليس الإشارة الوحيدة (StatusBadge = لون + نص + أيقونة).
- Focus rings 2px primary + 2px offset.
- `aria-live` على التقدّم غير المتزامن.
- Form errors: `aria-describedby` على كل input.
- Touch targets ≥ 44×44px.
- RTL focus mirroring طبيعي.

## ١١. Backend Gaps (يجب سدّها قبل M2)

اكتُشفت أثناء pre-flight على `main@093a082`. الواجهة تعتمد على هذه العقود، والـbackend لا يوفّرها بعد:

### ١١.١ M1 Blocker (Auth + Session)

| Endpoint متوقّع | الحالة | العقد المطلوب |
|---|---|---|
| `GET /auth/me` | غير موجود | `AuthController` يُضيف هذا. **العقد الكامل:**<br/>`{ user: { id, email, name, role }, onboardingCompleted: boolean, subscriptionStatus: 'trial' \| 'active' \| 'past_due' \| 'canceled' \| null, tenantId: string }`<br/>الـfrontend يحتاج `onboardingCompleted` لقرار التحويل dashboard ↔ onboarding، و `subscriptionStatus` لعرض لافتة الفوترة. |
| Cookie session + CSRF | غير مطبَّق | `SameSite=Lax` + `Secure` (production) + double-submit CSRF token + Origin/Referer validation. تفاصيل في القسم ٥. |

### ١١.٢ M2 Blockers (Engine HTTP surface)

| Endpoint متوقّع | الحالة | العقد المطلوب |
|---|---|---|
| `POST /content/generate` | غير موجود — `PipelineService` متاح داخلياً فقط | `EngineController` جديد يلفّ `pipeline.generateOne`. Request: `{ brandProfileId, platform, topic?, brief? }`. Response **async-ready** (واحد من شكلين):<br/>- إذا كان التوليد سريعاً: `{ status: 'completed', post: PostDetail }`<br/>- إذا كان سيتجاوز ثوانٍ: `{ status: 'queued', jobId: string }` ثم الواجهة تستعلم `GET /content/jobs/:jobId` حتى `status: 'completed'` ثم `post: PostDetail`.<br/>العقد يتّفق مع شكل `/content/month-plan` نفسه لتسهيل الـpolling logic الموحَّد. |
| `POST /content/month-plan` | غير موجود — `MonthPlanService` متاح داخلياً فقط | Endpoint يبدأ الـBullMQ job ويعيد `{ jobId }`. |
| `GET /content/month-plan/:jobId` | غير موجود | Endpoint لقراءة التقدّم: `{ status: 'running' \| 'completed' \| 'failed', done: number, total: number, posts: PostSummary[] }`. |

**ملاحظة عن الـmonth plan:** ليس blocker لـM2 الأول. الـM2 ينتهي عند single-post generation (Onboarding → Generate → Edit → Approve). الـmonth plan ينتقل إلى **M2.1** يُنفَّذ بعد نجاح الـM2 مباشرة. هذا يحرّر الـfrontend من انتظار endpoint `month-plan` كامل.

**الإجراء المطلوب قبل M2:** Tariq (backend engineer) يُسلِّم مجموعتين منفصلتين:
1. **`auth-session-hardening`** — يُضيف `GET /auth/me` + cookie session middleware + CSRF (double-submit) + Origin validation. يعتمد قبل M1 (لا تكتمل شاشة `/dashboard` بدونه).
2. **`engine-http-surface`** — يُضيف `EngineController` بـ`POST /content/generate` (async-ready) + `POST /content/month-plan` + `GET /content/jobs/:jobId` موحَّد لكلا الـendpoints. يعتمد قبل M2.

كل واحدة منها scope مستقل قبل الـfrontend milestone الخاص بها. هذه الاعتمادات الحقيقية، وليست تفاصيل يمكن تأجيلها.
